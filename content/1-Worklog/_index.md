**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Get familiar with AWS and its core services.](1.1-week1/)

**Week 2:** [Analyze and compile documentation on database migration solutions on the AWS platform.](1.2-week2/)

**Week 3:** [Gain a solid understanding of the EC2 architecture, load balancing mechanisms, and auto scaling strategies.](1.3-week3/)

**Week 4:** [Master IAM user management and deploy storage solutions using Amazon S3.](1.4-week4/)

**Week 5:** [Review and reinforce all foundational knowledge acquired on cloud computing, storage services, networking, and security in AWS.](1.5-week5/)

**Week 6:** [Conduct in-depth research on automated security tools to optimize monitoring and incident response for potential risks.](1.6-week6/)

**Week 7:** [Understand the concepts of Infrastructure as Code (IaC).](1.7-week7/)

**Week 8:** [Master various AWS storage solutions and the data lifecycle management process.](1.8-week8/)

**Week 9:** [Optimize system performance by implementing caching with Amazon CloudFront and optimizing database queries on Amazon RDS for PostgreSQL to improve page loading speed and API response time.](1.9-week9/)

**Week 10:** [Develop FlashLearn – a web application that supports English learning through flashcards, quizzes, battles, and a learning community.](1.10-week10/)

**Week 11:** [Complete the technical documentation.](1.11-week11/)

**Week 12:** [Finalize community interaction features.](1.12-week12/)
